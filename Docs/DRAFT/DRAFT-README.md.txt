КАТАЛОГ С ЭТАЛОННЫМИ И РАБОЧИМИ ПОСЛЕДОВАТЕЛЬНОСТЯМИ
====================================================

В данном каталоге представлены девелоперские версии эталонных и рабочих словарей, из которых собирается словарная база для тестов на проникновение. Использовать эти словари для самих тестов не рекомендуется, из-за их небольшого объёма и непрезентабельности. Скорее, эти словари представляют теоретический интерес, как шаблон для разработки более полных и практичных словарей. Автор с удовольствием присоединится к обсуждению этих словарей, а также принимает предложения по их улучшению.

ИЗГОТОВЛЕНИЕ РАБОЧИХ СЛОВАРЕЙ ИЗ ЭТАЛОННЫХ
==========================================

ИНСТРУМЕНТАРИЙ
--------------

Для изготовления рабочих словарей из эталонных может понадобиться программа TextPipe для сложной обработки текста. Для наших целей важно, что эта программа способна осуществлять следующие преобразования в пакетном режиме:

- Осуществлять изменения регистра символов (сразу всех или по шаблону);
- удалять дублирующие строки;
- сортировать строки как литералы и по словам, в том числе и на иностранных языках;
- проводить перекодировку символов;
- изменять знак окончания строки;
- осуществлять реверс строк текста.

С помощью этих инструментов можно будет создавать новые словари из шаблонов. Программа очень дорогая, ищите её пиратские Abandon версии.

Для транслитерации текста и смены раскладки клавиатуры можно использовать бесплатный пакет PuntoSwitcher. Для редактирования полученных словарей, их объединения и проверки используйте любой текстовый редактор. Под Windows это AkelPad и Notepad++, под Linux — Kate и GEdit. Использование консольных редакторов не приветствуется, поскольку они «бедны» плагинами и средствами визуализации, что очень важно при представлении словарной базы. В крайнем случае можно использовать abandon версию программы Multi-Edit for DOS.

Также для объединения файлов словарей можно использовать утилиты командной строки.

Для «декартового произведения» всех сторок в словаре автор рекомендует использовать программу combinator.exe (combinator.bin в Linux) из пакета hashcat-utils (смотри на гитхабе). Нужно также использовать другие программы из этого пакета и из пакта hcxtools (только для Linux).

Собственно, этих программных средств достаточно, чтобы составить словарь любой сложности из эталонных моделей.

Ссылки на скачивание автор не приводит. Всё это программное обеспечение можно найти в торрентах, на гитхабе и в менеджере пакетов chocolatey.

ФОРМАТ ЭТАЛОННЫХ СЛОВАРЕЙ
-------------------------

Все эталонные словари поставляются в кодировке UTF-8, с BOM, по одному слову (словарной фразе) в строке. Пробелы внутри строки не допускаются, в отличие от специальных символов. Окончание строк текста — стандартное для Windows. Словари представляются в текстовом формате без разметки (text/plan). Комментарии в файлах словаря запрещены.
